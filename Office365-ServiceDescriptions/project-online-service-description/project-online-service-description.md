# <a name="microsoft-project-service-description"></a>Description du service Microsoft Project

## <a name="overview"></a>Vue d’ensemble

Microsoft Project les applications suivantes pour répondre aux besoins de votre organisation en matière de gestion de projets et de travail :

- Project pour le web
- Project Online 
- Project Online client de bureau

Cet article vous aidera à comprendre quelles applications sont fournies dans chaque Project abonnement et quelles fonctionnalités chaque application offre.

### <a name="project-for-the-web"></a>Project pour le web

Project pour le Web est l’offre la plus récente de Microsoft pour le travail basé sur le cloud et la gestion de projet. Project web fournit des capacités de gestion de travail simples et puissantes pour répondre à la plupart des besoins et des rôles. Project gestionnaires et les membres de l’équipe peuvent utiliser Project pour le Web pour planifier et gérer le travail de toute taille.

Project pour le Web est construit sur la plate-forme Microsoft Power. La plate-forme power se compose de PowerApps, Power Automate, Power BI, et le Dataverse Microsoft. Project pour les données Web est stockée dans le Microsoft Dataverse. 

Project pour le Web est disponible via trois abonnements : Project (plan 1), Project (plan 3) et Project (plan 5). Les fonctionnalités [répertoriées Project pour les fonctionnalités Web](#project-for-the-web-features) n’interagissent qu’avec les données dans Project pour la boutique de données Web.

### <a name="project-online"></a>Project Online
 
Project Online est une solution en ligne flexible pour la gestion Project portefeuille (PPM) et le travail quotidien. Project Online de puissantes capacités de gestion de projet pour la planification, la hiérarchisation et la gestion de projets et d’investissements dans le portefeuille de projets &mdash; de presque n’importe où sur presque n’importe quel appareil. Project Online peut être utilisé par les administrateurs, les gestionnaires et les visualiseurs de portefeuille, les gestionnaires de projets et de ressources, et les chefs et les membres d'équipe.

Project Online est construit sur la plate-forme SharePoint, et il stocke des données dans le SharePoint de données.

Project Online est disponible via deux abonnements : Project (plan 3) et Project (plan 5). Les fonctionnalités [énumérées dans Project Online fonctionnalités](#project-online-features) n’interagissent qu’avec les données de la Project Online SharePoint de données.

La fonctionnalité membre de l’Project Online est disponible par le biais d’un abonnement Project Online Essentials ou Project (plan 1)'équipe. Project Online Essentials est un abonnement *add-on* membre de l’équipe pour les clients qui ont Project (plan 3) ou Project (plan 5) abonnements. Project Online Essentials et Project (plan 1) abonnements donnent aux membres de l’équipe une interface Web pour effectuer des mises à jour des tâches et du temps des projets gérés Project Online.

Les membres de l’Project Online Essentials ou Project (plan 1) abonnements peuvent effectuer les fonctions Project Online suivantes :

- Utiliser une interface web
- Mettre à jour des tâches, des problèmes et des risques
- Soumettre des feuilles de temps
- Partagez des documents et collaborez avec Microsoft Teams utilisateurs Skype Entreprise utilisateurs

Project Online Essentials n’inclut pas Project pour le web. Project (plan 1) inclut à la fois Project pour le web et Project Online Essentials.

### <a name="project-online-desktop-client"></a>Project Online client de bureau

Microsoft Project Online client de bureau est un programme de gestion de projet qui comprend la planification automatisée, la gestion des ressources de projet et les rapports intégrés. Project Online client de bureau peut être utilisé comme une application autonome ou il peut se connecter à Project Online. Project Online client de bureau est la version d’abonnement du client Project Professionnel bureau et est disponible en Project (plan 3) et Project (plan 5) abonnements.

### <a name="using-project-for-the-web-and-project-online-together"></a>Utilisation Project pour le web et Project Online ensemble

Bien Project pour le Web et Project Online sont des applications distinctes, vous pouvez les utiliser côte à côte. Vous pouvez voir tous vos projets à partir des deux applications dans Project Home. Vous pouvez utiliser la capacité Project feuille de route pour construire des feuilles de route qui incluent des projets à partir des deux applications. Et vous pouvez utiliser Power BI et les Project pour le pack de contenu web Power BI pour obtenir un aperçu des projets et des ressources à travers les deux applications. <sup>3</sup>

Ce diagramme montre comment les applications peuvent travailler ensemble côte à côte.

![Diagramme montrant Project sur le web et Project Online ensemble](../media/project-online-applications.png)

À l’avenir, Microsoft se concentrera sur Project pour les innovations web. Toutefois, vous pouvez continuer à utiliser Project Online confiance car il continuera à recevoir des améliorations clés de performance et de sécurité.

## <a name="service-considerations"></a>Considérations de service

> [!NOTE]
> Project pour le web n’est actuellement pas disponible pour Cloud de la communauté du secteur public, Cloud de la communauté du secteur public High et DoD. Nous travaillons sur la livraison de Project web pour vous, mais ne sont pas en mesure de partager un calendrier ferme pour le moment.

### <a name="licensing-considerations"></a>Remarques sur les licences

- Toute interaction sur un site Project Online nécessite au moins un abonnement Project (plan 3) ou Project (plan 5) au sein du locataire.
- Lorsque votre dernier abonnement Project (plan 1), Project (plan 3) ou Project (plan 5) expire, votre Project pour les instances Web ne sera pas automatiquement supprimée tant que vous n’aurez pas d’abonnements actifs dépendant de microsoft Dataverse.
- Lorsque votre dernier Project (plan 3) ou Project (plan 5) expire, vos instances Project Online seront supprimées après 120 jours.
- Pour Project Online abonnements d’essai, vos instances d’essai seront supprimées 30 jours après la fin de votre période d’essai.
- Par Project abonnements à l’essai web, vos instances d’essai ne seront pas supprimées tant que vous n’aurez pas d’abonnements actifs dépendant de microsoft Dataverse.

### <a name="use-of-sharepoint-online"></a>Utilisation de SharePoint Online

Project Online nécessite l’utilisation de SharePoint online, qui est prévu dans le cadre de Project Online. Les droits sur les SharePoint en ligne fournies avec les abonnements Project (plan 3) ou Project (plan 5) sont limités au stockage et à l’accès aux données pour prendre en charge Project Online.

### <a name="project-roadmap-and-power-automate"></a>Project Feuille de route et Power Automate

Project La feuille de route nécessite l’utilisation Power Automate, qui est prévu dans le cadre de votre abonnement Project’abonnement. Les droits Power Automate fonctionnalités sont limités aux capacités Power Automate requises par la feuille Project feuille de route. Power Automate fonctionnalité requise par Project Roadmap apparaît comme **l’intégration de données pour Project avec** un plan de service Flow dans la section **Apps** du Microsoft 365 admin center.

### <a name="project-for-the-web-and-microsoft-dataverse"></a>Project pour le web et Microsoft Dataverse

Project pour le Web nécessite l’utilisation de microsoft Dataverse pour stocker ses données. Une base de données Microsoft Dataverse est mise à disposition dans le cadre de Project abonnement. Les droits sur les fonctionnalités Microsoft Dataverse se limitent au stockage et à l’accès aux données pour prendre en charge Project pour le Web. La fonctionnalité Microsoft Dataverse requise par Project apparaît comme **le service de données commun pour** un plan de service Project dans la section **Apps** du Microsoft 365 d’administration.

Pour Project clients ayant cinq (5) ou plus Project pour les licences Web, les Project pour les clients Web peuvent être déployées dans les environnements Power Platform Production et Sandbox.
